术语速查 ​

术语本站中的含义
rollout用策略/LLM server 对 prompt 生成一条或多条 trajectory
trajectoryprompt、response、mask、概率、reward 与环境元数据组成的轨迹
actor被优化的策略模型训练角色
reference冻结策略，用于 KL 锚定
critic预测 value 的模型/角色
reward manager把 response 与 ground truth/模型/环境反馈转成 score
advantage相对基线的策略更新信号
staleness生成轨迹所用权重与当前训练权重之间的版本延迟
TQTransferQueue，V1 的 trajectory KV 数据通道
KVBatchMeta指向 TQ key/tag 的轻量批次句柄
DataProtoveRL 的批次协议对象；V0 主干，V1 部分边界仍使用
WorkerGroup对一组 Ray worker/rank 统一分发计算的控制抽象

内容边界 ​

本站重点是默认 V1 的 PPO/GRPO 学习主线，没有穷举所有模型引擎、rollout backend、多模态 agent、蒸馏、完全异步 recipe 和硬件组合。以下情况必须回到对应提交的源码/测试：

  • 改用 trainer.use_v1=false；
  • 使用自定义 AgentLoop/ReplayBuffer/RewardManager；
  • 使用非默认 policy loss 或 rollout correction；
  • 采用 Megatron/VeOmni/TorchTitan、MoE、NPU 或多机 RDMA；
  • 升级 commit 后类名、字段或默认配置发生变化。

发现本站与绑定提交不一致时，以可执行测试和源码为准，并在修正文档时同时记录旧/新 commit。
