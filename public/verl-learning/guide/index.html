学习路线 ​

这套材料的目标不是把 veRL 的每个文件翻译一遍，而是让你能回答四个问题：一次样本怎样生成、奖励怎样落到 token、优势怎样变成梯度、模型新权重怎样回到推理引擎。

IMPORTANT

本站绑定 veRL 提交 e5687fce，默认讲解 trainer.use_v1=true 的新训练器。阅读其他版本时，先看版本边界。

一张图看完 ​

四个阶段 ​

1. 建立最小理论模型 ​

先读 LLM 与后训练 和 强化学习基础。你不需要先学完一门深度学习课程，但需要知道：

  • 模型给出的不是“文字”，而是下一个 token 的概率分布；
  • rollout 是按策略采样轨迹，训练是对已有轨迹重新计算概率并反向传播；
  • reward 是结果信号，advantage 才是“这次动作比基线好多少”的训练信号；
  • attention mask 与 response mask 决定哪些 token 真正参与统计和损失。

检查点：能用自己的话解释 response、old_log_prob、ref_log_prob、values、advantages 的区别。

2. 用算法给字段赋义 ​

接着读 PPO 与 GAE 和 GRPO 与算法家族。此时公式只服务于源码：看到 ratio、clip、KL 或 group normalization，能立刻知道输入字段和输出字段。

检查点：能解释为什么 rollout 通常要保留行为策略的概率信息，以及 GRPO 为什么要求同一 prompt 生成多个回答。

3. 沿一次 step 读源码 ​

不要从目录第一行开始。按下面的顺序阅读：

  1. 整体架构：先确认进程、资源与数据通道；
  2. 入口与初始化：找到对象何时建立；
  3. 从推理到训练：跟踪一批数据；
  4. 数据结构与组织：补齐 TensorDict、TransferQueue、KVBatchMeta 与 DataProto；
  5. Worker 与资源编排：理解 Ray 角色和 GPU 放置。

检查点：能从 main_ppo.py 追到 actor 更新，并指出数据在哪一步只传句柄、在哪一步真正被 worker 读取。

4. 做一个可复现实验 ​

按 配置方法 建立最小配置，按 奖励函数 写纯函数和单元测试。遇到问题先用 调试与排障 定位阶段，再用 性能采集 证明瓶颈。

检查点：保存完整命令、解析后的配置、代码提交、环境版本、训练日志和关键指标；换机器后仍能说明实验差异来自哪里。

推荐的阅读姿势 ​

在编辑器中同时打开网页与源码。每读完一节，做三件小事：画出输入输出、在源码中找到对应函数、用一条日志或断言验证自己的判断。只看类名很容易形成“好像懂了”的错觉；跟踪一条真实字段才会暴露边界条件。
