强化学习基础：把语言生成映射成决策过程 ​

在 LLM RL 中，一条回答可以看成一条轨迹：状态是已有上下文，动作是下一个 token，策略是语言模型，episode 在 EOS 或长度上限结束。

对应关系 ​

强化学习术语LLM 后训练中的含义
state stprompt 加上已生成 token
action at第 t 个 response token
policy πθ参数为 θ 的语言模型
trajectory τ一条完整生成序列及其元数据
reward rt规则、奖励模型或环境返回的分数
return Gt从当前位置开始的折扣累计奖励
value V(st)critic 对未来 return 的估计
advantage At该动作相对基线好多少

从 reward 到 advantage ​

回报定义为：

Gt=∑l=0T−t−1γlrt+l.

直接用 Gt 更新策略方差通常较大。减去不依赖当前动作的基线不会改变期望梯度，却能降低方差：

At=Gt−V(st).

直观上，reward 说“最终得了 1 分”，advantage 说“相对于这类状态通常能得到的分数，这个 token 所在轨迹更好还是更差”。正优势提高已采样动作概率，负优势降低它。

GAE 用 temporal-difference residual 在偏差与方差之间折中：

δt=rt+γV(st+1)−V(st),A^t=∑l≥0(γλ)lδt+l.

当奖励只放在回答末尾时，mask、终止位置和 bootstrapping 是否正确，直接决定信号能否传回前面的 token。

Policy gradient 的核心直觉 ​

策略梯度最简形式是：

∇θJ(θ)=E[∑t∇θlog⁡πθ(at∣st)At].

它并不对离散 token 本身求导，而是对“模型给已采样 token 的对数概率”求导。PPO/GRPO 等方法主要改变优势如何估计、旧数据如何受约束、损失如何聚合。

On-policy、off-policy 与 staleness ​

  • on-policy：轨迹由正在优化的策略产生；理论最干净，但生成成本高。
  • off-policy：轨迹来自不同策略，需要重要性比率或其他校正。
  • staleness：异步系统中，轨迹生成期间 actor 已更新多个版本，是工程上常见的 off-policy 来源。

veRL V1 的 replay buffer 会记录轨迹跨越的权重版本，并按 max_off_policy_threshold 与 drop/wait 策略处理过旧样本。这个机制解决的是调度层面的新鲜度控制；训练侧仍可能配置 rollout correction。

KL 为什么出现 ​

只追 reward 可能把策略推向狭窄、投机或语言质量差的区域。参考策略 πref 提供锚点，常见约束基于：

DKL(πθ∥πref).

代码中要区分两条路径：把 KL 作为 reward penalty，或在 actor loss 中加入 KL loss。两者的配置、缩放和统计口径不同，不要同时打开后又误以为只约束了一次。

Credit assignment 的两个层次 ​

结果奖励常只告诉整条回答对错，却没有指出哪一步推理造成结果。优势估计解决“数值怎样分配到 token”的一部分问题，但不会自动创造细粒度监督。过程奖励、环境的逐步反馈、多轮工具轨迹或验证器可以提供更细信号，同时也引入对齐、延迟与安全边界。
