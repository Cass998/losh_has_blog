GRPO 与相关算法：先看基线，再看配方 ​

GRPO 的工程吸引力在于：同一个 prompt 采样多个回答，用组内相对分数构造 advantage，不需要单独训练 critic。代价是每个 prompt 的生成次数增加，而且组内全对或全错时学习信号可能消失。

标准 GRPO 的组内优势 ​

对 prompt 组 g 的 G 个回答，先得到每条回答的 outcome reward ri：

μg=1G∑i=1Gri,σg=1G−1∑i=1G(ri−μg)2.

veRL 当前 compute_grpo_outcome_advantage 的核心是：

Ai=ri−μgσg+ϵ,

再把这个序列级标量广播到该回答的每个有效 token。配置 algorithm.norm_adv_by_std_in_grpo=false 时只减均值、不除标准差，对应源码注释中的 Dr.GRPO 形式。

在配置上，actor_rollout_ref.rollout.n 决定一个 prompt 生成多少候选；data.train_batch_size 通常以原始 prompt 数计。读日志时要分清 prompt 数、response 数和有效 token 数。

边界情况 ​

  • 组大小为 1：当前非向量化实现把均值设为 0、标准差设为 1；这不再具有“相对组基线”的通常含义。GRPO 实验应明确使用 rollout.n > 1。
  • 组内奖励完全相同：减均值后 advantage 为 0，没有策略梯度信号；增大组大小不保证能解决奖励饱和。
  • 无效或缺失轨迹：多轮 AgentLoop 可能返回空结果。V1 工具代码只用每个 session 的最终输出参与 GRPO，再把结果散射回同 session 的其他输出。
  • 长度偏好：序列级 advantage 被广播到 token 后，最终影响仍受 loss aggregation 和 response 长度影响。

不要把“算法配方”都当枚举值 ​

当前 AdvantageEstimator 注册了 gae、grpo、reinforce_plus_plus、reinforce_plus_plus_baseline、remax、rloo、opo、grpo_passk、gpg、向量化变体、token baseline 变体和 gdpo。下面的名称处在不同层次：

名称核心基线/信号在当前 veRL 中怎样落地
GRPO同 prompt 组均值与标准差adv_estimator=grpo，通常 rollout.n>1
Dr.GRPO组均值，不除组标准差grpo + norm_adv_by_std_in_grpo=false
RLOOleave-one-out：其余候选均值作基线adv_estimator=rloo 或向量化变体
ReMax贪心回答的 reward 作 baselineadv_estimator=remax；rollout 需额外 baseline 生成
REINFORCE++return/advantage 的特定归一化配方对应已注册 estimator，核对函数参数与 mask
GDPO多奖励维度分别组内归一化后加权adv_estimator=gdpo + reward keys/weights
DAPO含动态采样、clip/loss/reward 等的一套训练配方不是 AdvantageEstimator.DAPO；由多项配置和 reward manager 组合

同一个论文名可能跨越 advantage、reward、sampling 与 policy loss 四层。判断实现是否“支持某算法”，要逐层核对，不能只搜索一个字符串。

GRPO 与 PPO 的关系 ​

GRPO 常被描述为“不用 critic 的 PPO 变体”。在工程里可以这样拆：

GRPO 主要替换 advantage estimator；旧策略约束、KL、loss aggregation、off-policy correction 等仍是独立选择。adv_estimator=grpo 不意味着所有其他部分都自动采用某篇论文的完整超参数。

一组可验证的配置意图 ​

yaml
algorithm:
  adv_estimator: grpo
  norm_adv_by_std_in_grpo: true

actor_rollout_ref:
  rollout:
    n: 8

critic:
  enable: false

这段只表达意图，不是可以脱离完整 Hydra 组合单独运行的配置。实际运行前检查：reward 是否在同组内有区分度、batch 是否包含完整 prompt group、critic 是否确实被条件关闭、每个有效 response 是否拥有一致的 group/index 标识。
